# ТЗ: окно тестирования моделей в LabelHub

Дата согласования: 2026-06-14.

## 1. Решения (зафиксировано с заказчиком)

| Вопрос | Решение |
|---|---|
| Навигация | В шапке две кнопки-вкладки: **«Датасеты»** (нынешний `ProjectList`, переименование) и **«Модели»** (новый экран) |
| Источник плашек | Завершённые run'ы (есть `best.pt`) **+** импортированные `.pt` |
| Импорт | Выбор `.pt` с диска, **без привязки** к проекту, имена классов читаются из самого файла |
| Рендер инференса | **Стрим кадров в приложение** (реал-тайм, MJPEG) |
| Источники видео | Файл с диска · видео проекта · веб-камера |
| Контролы | conf-слайдер · play/pause/перемотка · сохранение размеченного видео; **выбора устройства нет** — показываем какое используется, по умолчанию GPU |
| Действия на плашке | Тестировать · Метрики/инфо · Переименовать · Удалить · Экспорт весов |

## 2. Модель данных

Новая таблица `models` — единый реестр всех моделей:

```
id            int PK
name          str        # редактируемое отображаемое имя
kind          str        # "trained" | "imported"
run_id        int?  FK training_runs   # для trained
project_id    int?       # проект-источник (trained); null для imported
weights_path  str        # абсолютный путь к .pt
base_model    str
imgsz         int  = 640
classes_json  str        # имена классов (читаются из YOLO(...).names)
map50         float?     # копия из run; null для imported
map5095       float?
created_at    datetime
```

`init_db` (`create_all`) подхватит новую таблицу автоматически.

**Ленивая синхронизация:** `GET /api/models` проходит по `training_runs` со `status="completed"`,
у которых существует `{run_dir}/weights/best.pt`, и создаёт недостающие строки `kind="trained"`
(чтобы уже обученные run'ы появились без переобучения). Для каждой модели проверяет наличие файла
весов и помечает отсутствующие (`missing: true`).

## 3. Бэкенд

`backend/api/models.py` — реестр:
- `GET /api/models` — список (с ленивой синхронизацией)
- `POST /api/models/import` — приём `.pt` (путь с диска), копирование в `data/models/imported/{uuid}.pt`,
  чтение `names`/task через `YOLO(path)`. Валидация: модель должна быть OBB (`task == "obb"`)
- `PATCH /api/models/{id}` — переименование
- `DELETE /api/models/{id}` — удаление строки реестра; для `imported` — ещё и удаление .pt; для
  `trained` сам run не трогаем
- `GET /api/models/{id}/export` — отдать `best.pt` на скачивание

`backend/api/inference.py` — сессии инференса:
- `POST /api/inference/start` `{ model_id, source }` (source: file/project_video/webcam) → `session_id`.
  Активна одна сессия за раз
- `GET /api/inference/{sid}/stream` — MJPEG `multipart/x-mixed-replace`
- `POST /api/inference/{sid}/control` `{ action: play|pause|seek|conf|record, value }`
- `GET /api/inference/{sid}/status` — `{ frame, total, fps, device, detections, model_name, finished, output }`
- `POST /api/inference/{sid}/stop`
- `GET /api/inference/{sid}/download` — скачать записанное размеченное видео

`backend/services/inference_service.py` — менеджер сессий. Каждая сессия = фоновый поток:
`cv2.VideoCapture` → `model(frame)` → отрисовка OBB → JPEG в общий буфер, который читает MJPEG-генератор.
Флаги `paused`/`seek`/`conf`/`record` читаются потоком вживую. Устройство: `0` если CUDA доступна,
иначе `cpu`; фактическое значение отдаётся в `/status`.

## 4. Фронтенд

- Шапка `App.jsx`: навигация с вкладками «Датасеты» / «Модели». Роуты: `/` → `ProjectList`,
  `/models` → `ModelList`.
- `ModelList.jsx` — грид плашек: имя, бейдж обучена/импортирована, базовая модель, mAP50/mAP50-95,
  проект-источник, дата, число классов. Действия: Тестировать · Метрики/инфо · Переименовать · Удалить ·
  Экспорт весов. Кнопка «Импортировать модель».
- `ModelTester.jsx` — модалка: выбор источника (Файл / Видео проекта / Веб-камера), `<img>` с MJPEG,
  контролы (conf, play/pause, перемотка, стоп, запись), статус-строка (устройство, кадр X/Y, FPS,
  детекции, имя модели).
- `api/client.js` — методы models + inference.

## 5. Electron

- Импорт `.pt`: IPC-хендлер выбора `.pt` + проброс в preload.
- Экспорт весов: обобщить `select-save-location` (принимать расширения), чтобы сохранять `.pt`/`.mp4`.

## 6. Риски / на что заложиться

- Веб-камера открывается на бэкенде (`cv2.VideoCapture(index)`) — подходит для локального запуска.
- Импортированный `.pt` может быть не-OBB → отклоняем с понятной ошибкой.
- Освобождение ресурсов: закрытие модалки/смена источника → `stop` (иначе зависший захват камеры).
- Производительность: на CPU низкий FPS — статус-строка показывает реальный FPS и устройство.
